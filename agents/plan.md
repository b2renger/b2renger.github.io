# Comprehensive Implementation Roadmap: b2renger Portfolio SPA

This document serves as the master implementation plan to modernize and simplify the architecture of the portfolio while preserving its framework-free nature.

## ✅ Phase 1: Data Architecture & Markdown Migration (Completed)
**Goal:** Flatten the data structure, introduce chronological sorting, and simplify content authoring via Markdown.

- [x] **New JSON Architecture:** Convert the `content` property from an array of blocks into a single Markdown string in `data.js`.
- [x] **Chronological Context:** Add a `year` attribute to all projects. Sort the project grid by year automatically.
- [x] **Data Migration:** Cross-reference `data.js` objects with the original `CSV` files in the master git history and translate legacy values into standard Markdown syntax.
- [x] **Rendering Engine:** Include `marked.js` in `index.html` via CDN. Update `showProject()` in `script.js` to parse `project.content` directly to `innerHTML`.

---

## ✅ Phase 2: Hash Routing (Deep-linking) (Completed)
**Goal:** Fix browser navigation (Back/Forward buttons) and enable direct links to specific projects without requiring a backend or History API.

- [x] **URL Updating Logic:** 
   - When a user clicks a project (in `showProject()`), update the browser's URL hash: `window.location.hash = '#project/' + project.id`.
   - When a user clicks "Back to Portfolio" (in `showHome()`), update the URL hash: `window.location.hash = ''` (or `#home`).
- [x] **Hash Change Listener:**
   - In `script.js` inside the `DOMContentLoaded` block, append a `window.addEventListener('hashchange', routerLogic)`.
   - Create a `routerLogic()` function that inspects `window.location.hash`.
     - If it matches `#project/<id>`, find the project and call `showProject(id)` bypassing the state push.
     - If it's empty or `#home`, call `showHome()`.
- [x] **Bootstrapping:** 
   - Modify `app.init()` so that instead of always booting directly to `showHome()`, it evaluates the current URL hash on load and correctly drops the user straight into the requested project.

---

## ✅ Phase 3: Media Expansion & Markdown Styling (Completed)
**Goal:** Ensure the newly generated Markdown safely and beautifully renders advanced media embeds, audio, and code.

- [x] **CSS Styles for Markdown (`style.css`):**
   - Provide standard styling rules for `<div class="detail-content">` child elements.
   - Headers: `h1, h2, h3`
   - Lists: `ul, ol, li`
   - Typography: Add formatting to `blockquote` or `strong/em` tags.
- [x] **Media Embeds CSS:**
   - Standardize responsive `<iframe class="video-container">` wrappers so YouTube/Vimeo embeds spawned from Markdown are properly constrained on mobile devices. Note: Utilized modern `aspect-ratio` CSS attribute.
   - Apply styling for `<audio>` elements generated by Markdown code blocks or standard HTML inserts.
- [x] **Syntax Highlighting (Completed with pure CSS):**
   - Custom styled `pre code` tags utilizing `--color-tag-bg` natively instead of heavy libraries.

---

## 🛠 Phase 4: Data Polish & Cleanup
**Goal:** Give full control back to the user to comfortably author new projects.

1. **Review Migrated CSVs:** The user opens `data.js` to verify, fix, and style the migrated copy.
2. **Module System Shift (Optional):** We can further clean the global namespace by switching `script.js` to `<script type="module">` and dropping `const projects = ...` in favor of `export const projects = ...`.
